## Phase 1: Noteコンテンツ制作ワークフロー

### タスク1: Note記事ネタ調査

**目的**: トレンドと既存記事ラインナップを分析し、執筆すべき記事ネタを10種類程度リスト化する

**現状の課題**:
- 所要時間: 約30分
- AIエージェント（Manus/GenSpark）のチャット欄のみに記録される
- GitHubなどでの管理ができていない

**自動化案**:

| 工程 | ツール | 処理内容 | 自動化レベル |
| :--- | :--- | :--- | :--- |
| トレンド調査 | **Manus/GenSpark** | 最新のAI/Tech業界トレンドをリサーチ | 完全自動 |
| 既存記事分析 | **Manus/GenSpark** | note.com/instkoniの既存記事を分析し、カバーしていない領域を特定 | 完全自動 |
| ネタリスト生成 | **Manus/GenSpark** | 10種類の記事ネタを提案 | 完全自動 |
| 構造化保存 | **Claude Code Desktop** | 提案をJSON/Markdown形式でGitHubリポジトリに自動保存 | 完全自動 |

**ワークフロー**:
```
[Manus/GenSpark] トレンド調査 + 既存記事分析
    ↓
[Manus/GenSpark] 記事ネタ10種類提案
    ↓
[Claude Code Desktop] JSON/Markdown形式で保存
    ↓
[GitHub] リポジトリに自動コミット（/ideas/YYYY-MM-DD_ideas.md）
```

**成果物**: `/ideas/2026-01-16_ideas.md`（GitHubで管理）

**最適化ポイント**:
- 提案を構造化データとして保存することで、後続の分析や優先順位付けが容易になる
- GitHubでバージョン管理することで、過去の提案を追跡可能

### タスク2: Note執筆

**目的**: 選定された記事ネタについて、4000文字程度の記事を執筆し、NanobananaProでインフォグラフィックを作成する

**現状の課題**:
- 記事ネタ選定: 約10分
- 調査・情報収集: 約20分
- 情報精読: 約20分（人間）
- 記事作成: 約10分
- インフォグラフィック作成: 約20分（デザインにムラがある）
- Noteへの貼り付け・修正: 約60分
- 画面キャプチャ取得: 約10分
- 記事精査: 約30分（人間）
- トップページ作成（Canva）: 約30分
- タグ壁打ち: 約10分
- 公開: 約10分

**自動化案**:

| 工程 | ツール | 処理内容 | 自動化レベル |
| :--- | :--- | :--- | :--- |
| 記事ネタ選定 | **Manus/GenSpark** | 保存された10種類から、エンゲージメント予測に基づき選定 | 完全自動 |
| 深層リサーチ | **Manus/GenSpark** | 対象サービス/技術の調査、SNS発信状況の分析 | 完全自動 |
| 記事執筆 | **Manus/GenSpark** | note.com/instkoniのスタイルを学習し、4000文字の記事を生成 | 完全自動 |
| インフォグラフィック作成 | **GenSpark + NanobananaProまたはManus** | 統一デザインテンプレートを使用してインフォグラフィックを生成 | 完全自動 |
| 画面キャプチャ取得 | **Claude Code Desktop** | 対象サービスのスクリーンショットを自動取得 | 完全自動 |
| トップページ作成 | **Claude Code Desktop + Canva API** | Canvaテンプレートを使用して自動生成 | 半自動 |
| 記事保存 | **Claude Code Desktop** | Markdown形式でGitHubに保存 | 完全自動 |
| タグ生成 | **Manus/GenSpark** | 記事内容に基づき最適なタグを提案 | 完全自動 |
| Note投稿 | **手動** | 人間が最終確認して投稿 | 手動 |

**ワークフロー**:
```
[Manus/GenSpark] 記事ネタ選定 + 深層リサーチ
    ↓
[Manus/GenSpark] 記事執筆（4000文字、既存スタイル適用）
    ↓
[GenSpark/Manus + NanobananaProまたはManus] インフォグラフィック生成
    ↓
[Claude Code Desktop] 画面キャプチャ自動取得
    ↓
[Claude Code Desktop + Canva API] トップページ生成
    ↓
[Claude Code Desktop] Markdown形式でGitHub保存（/articles/YYYY-MM-DD_title.md）
    ↓
[Manus/GenSpark] タグ提案
    ↓
[人間] 最終確認 + Note投稿
```

**成果物**: 
- `/articles/2026-01-16_article.md`（記事本文）
- `/assets/infographics/2026-01-16_infographic.png`（インフォグラフィック）
- `/assets/covers/2026-01-16_cover.png`（トップページ）

**最適化ポイント**:
- インフォグラフィックのデザインテンプレートを事前に定義し、統一感を確保
- GitHubで記事をバージョン管理することで、修正履歴を追跡可能
- Canva APIを活用することで、トップページ作成を自動化

**ワークフロー**:
```
[Claude Code Desktop] プロンプト・歌詞を読み込み
```

| 工程 | ツール | 処理内容 | 自動化レベル |
| :--- | :--- | :--- | :--- |
| リップシンクなし動画生成 | **Claude Code Desktop** | Higgsfield/Vidu AIで画像+プロンプトから動画生成 | 完全自動 |
| リップシンク動画生成 | **Claude Code Desktop** | Heygenで画像+vocals.mp3+感情プロンプトから生成 | 完全自動 |
| 人間確認 | **人間** | 生成された動画を確認し、使用するクリップを選択 | 手動 |
| ダウンロード | **Claude Code Desktop** | 選択されたクリップをダウンロード | 完全自動 |

**ワークフロー**:
```
[Claude Code Desktop] プロンプト + 画像を読み込み
    ↓
【リップシンクなし】
[Claude Code Desktop] Higgsfield/Vidu AIブラウザ操作で生成
    ↓
【リップシンクあり】
[Claude Code Desktop] Heygenブラウザ操作で生成（画像+vocals.mp3+感情プロンプト）
    ↓
[人間] 動画確認 → 使用するクリップを選択
    ↓
[Claude Code Desktop] 選択クリップをダウンロード
    ↓
[Claude Code Desktop] /mv_projects/PROJECT_NAME/videos/raw/ に保存
```
