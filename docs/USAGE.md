# 使い方ガイド

このドキュメントでは、SNSコンテンツ自動化プロジェクトの具体的な使い方を説明します。

---

## 🚀 はじめに

このプロジェクトは、Note記事作成からマルチプラットフォーム配信までのワークフローを効率化・自動化するために設計されています。

**主な目的:**
- AIエージェントで生成した情報をGitHubで一元管理
- 記事作成プロセスの可視化と標準化
- 各プラットフォーム向けコンテンツの効率的な生成

---

## 📝 基本的な使い方

### 1. 新しい記事アイデアの追加

記事のアイデアが浮かんだら、まず`ideas/ideas-list.md`に記録します。

**手順:**
1. `ideas/ideas-list.md`を開く
2. 「🆕 新規アイデア」セクションに新しい行を追加
3. 記事IDを付与(連番)
4. タイトル案、カテゴリ、優先度、メモを記入
5. ファイルを保存してコミット

**例:**
```markdown
| 002 | 2026-01-15 | AIで副業を効率化する方法 | 副業 | 高 | 実体験ベース |
```

---

### 2. 調査・リサーチの実施

記事ネタが決まったら、調査を実施します。

**手順:**
1. `templates/research-template.md`をコピー
2. `research/topics/research-XXX.md`として保存(XXXは記事ID)
3. AIエージェント(Genspark/Manus)で調査実施
4. テンプレートに沿って調査結果を記録
5. SNS発信状況も併せて記録
6. ファイルを保存してコミット

**AIエージェントへの指示例:**
```
「AIで副業を効率化する方法」というテーマで、以下を調査してください:
1. 最新のトレンドと事例
2. 主要なツールとサービス
3. TwitterとLinkedInでの議論状況
4. 参考になる記事やブログ
```

---

### 3. 記事の執筆

調査が完了したら、記事を執筆します。

**手順:**
1. `templates/article-template.md`をコピー
2. `articles/drafts/YYYY-MM-DD-title.md`として保存
3. 調査結果を参照しながら記事を執筆
4. AIエージェントで初稿を生成してもOK
5. メタ情報(キーワード、ハッシュタグ)も記入
6. ファイルを保存してコミット

**AIエージェントへの指示例:**
```
以下の調査結果をもとに、Note記事を作成してください:
[research-XXX.mdの内容を貼り付け]

条件:
- 読了時間: 5-7分
- ターゲット: 副業を始めたい会社員
- トーン: 親しみやすく、実践的
- 構成: イントロ、3つのセクション、まとめ
```

---

### 4. インフォグラフィックの作成

記事に添付するインフォグラフィックを作成します。

**手順:**
1. Gensparkでインフォグラフィックを生成
2. デザインの統一性を確認(色、フォント、キャラクター)
3. `assets/infographics/infographic-XXX.png`として保存
4. 記事テンプレートの「ビジュアル素材」セクションに情報を記入
5. ファイルをコミット

**デザインガイドライン:**
- カラーパレット: [統一する色を定義]
- フォント: [統一するフォントを定義]
- キャラクター: [統一するキャラクターを定義]

---

### 5. Note投稿の準備

記事とビジュアル素材が揃ったら、Note投稿の準備をします。

**手順:**
1. 記事本文をNoteプラットフォームにコピー
2. インフォグラフィックをアップロード・配置
3. 引用、コード、文字の修正
4. 必要な画面キャプチャを追加
5. CANVAでトップ画像を作成
6. トップ画像をアップロード
7. タグを設定
8. プレビューで最終確認

---

### 6. Note記事の公開

**手順:**
1. 公開設定を確認
2. 公開実行
3. 公開URLを記事テンプレートに記録
4. `articles/drafts/`から`articles/published/`に移動
5. `ideas/ideas-list.md`のステータスを「✅完了」に変更
6. コミット

---

### 7. LinkedIn配信

Note記事をLinkedIn向けにリライトして投稿します。

**手順:**
1. Geminiで記事をLinkedIn向けにリライト
2. `articles/published/linkedin-XXX.md`として保存
3. LinkedInに投稿
4. インフォグラフィックを添付
5. タグ付け、絵文字調整
6. 投稿URLを記事テンプレートに記録
7. コミット

---

### 8. X(Twitter)配信

Note記事をX向けにパーツ分解してスレッド投稿します。

**手順:**
1. 記事を3-4個のポストに分解
2. 各ポストにハッシュタグを設定
3. `articles/published/x-thread-XXX.md`として保存
4. Xにスレッド投稿
5. インフォグラフィックを添付
6. 投稿URLを記事テンプレートに記録
7. コミット

## 🔄 Git操作の基本

### ファイルの追加とコミット

```bash
# 変更を確認
git status

# ファイルを追加
git add .

# コミット
git commit -m "記事XXX: [作業内容の説明]"

# GitHubにプッシュ
git push origin main
```

### コミットメッセージの例

- `記事001: 新規アイデア追加`
- `記事001: 調査完了`
- `記事001: 記事執筆完了`
- `記事001: Note公開、LinkedIn/X投稿完了`

## 📊 進捗管理

### 日次レビュー

毎日の作業終了時に以下を確認:
- `workflows/workflow-checklist.md`の更新
- 完了したタスクのチェック
- 翌日のタスク確認

### 週次レビュー

毎週末に以下を実施:
- 今週作成した記事数の確認
- 各プラットフォームのパフォーマンス確認
- 改善点の洗い出し

### 月次レビュー

毎月末に以下を実施:
- 月間目標の達成状況確認
- エンゲージメント分析
- ワークフローの改善検討

## 🛠️ トラブルシューティング

### よくある問題と解決方法

**問題1: AIエージェントの出力が期待と異なる**
- 解決策: プロンプトをより具体的に調整する
- 解決策: 調査結果をより詳細に提供する

**問題2: インフォグラフィックのデザインが統一されない**
- 解決策: デザインガイドラインを明確に定義する
- 解決策: Gensparkに統一デザインを指示する

**問題3: Git操作でエラーが発生**
- 解決策: `git status`で状態を確認
- 解決策: コンフリクトがある場合は手動で解決

## 💡 効率化のヒント

1. **テンプレートの活用**: 毎回同じ構造で作成することで、作業時間を短縮
2. **AIプロンプトの保存**: 効果的なプロンプトは記録して再利用
3. **バッチ処理**: 複数の記事を同時進行で効率化
4. **定期的なレビュー**: ワークフローを継続的に改善
