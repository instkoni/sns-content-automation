# テーマファイル作成支援 Skill

このSkillは、ユーザーと対話しながら、記事作成用のカスタムテーマファイルを作成します。

## ❗️ 重要な実行ルール

- **対話的に進める**: ユーザーの意図を確認しながら、段階的にテーマファイルを作成してください。
- **ウェブ検索を活用**: 最新のトレンドや関連情報を調査して、質の高いサブテーマを提案してください。
- **柔軟に対応**: ユーザーのフィードバックに応じて、内容を修正してください。

## タスクの流れ

### ステップ1: メインテーマのヒアリング

ユーザーに以下を質問してください。

```
カスタムテーマファイルを作成します。以下の質問に答えてください。

1. **メインテーマは何ですか？**
   （例: AIを活用した音楽制作、クリエイターエコノミーの最新動向、NFTアートの実践ガイド）

2. **どのような読者を想定していますか？**
   （例: 音楽クリエイター、デジタルアーティスト、副業を考えているビジネスパーソン）

3. **何本の記事を作成したいですか？**
   （例: 3本、5本、10本）

4. **特に重視したいポイントや、含めたいキーワードはありますか？**
   （例: 実践的な方法、最新ツールの比較、法律・著作権の解説）
```

### ステップ2: 最新トレンドの調査

ユーザーから回答を受け取ったら、以下を実行してください。

1. **ウェブ検索でメインテーマの最新トレンドを調査**
   - 現在の時間（2026年1月）の最新情報を収集
   - 話題になっているツール、サービス、事例を調査
   - クリエイターコミュニティでの議論を確認

2. **関連キーワードをリストアップ**
   - メインテーマに関連する重要なキーワードを10個程度抽出

### ステップ3: サブテーマ（記事タイトル案）の提案

調査結果を基に、以下の形式でサブテーマを提案してください。

```
【提案】以下のサブテーマ（記事タイトル案）を作成しました。

1. 【タイトル案1】
   - 概要: （このタイトルで何を伝えるか）
   - 狙い: （読者にどんな価値を提供するか）

2. 【タイトル案2】
   - 概要: 
   - 狙い: 

...

これらのタイトル案で問題ありませんか？
修正や追加のご要望があれば、お知らせください。
```

### ステップ4: ユーザーのフィードバック反映

ユーザーからのフィードバックを受け取り、以下を実行してください。

- タイトル案の修正
- 新しいタイトル案の追加
- 不要なタイトル案の削除

ユーザーが満足するまで、このステップを繰り返してください。

### ステップ5: テーマファイルの作成

ユーザーが承認したら、以下の形式で`current_theme.md`を作成してください。

```markdown
# カスタムテーマ: {{メインテーマ}}

## メインテーマ
{{メインテーマの詳細説明}}

## ターゲット読者
{{想定読者の説明}}

## サブテーマ（記事タイトル案）

1. {{タイトル案1}}
2. {{タイトル案2}}
3. {{タイトル案3}}
...

## 参考情報

- {{調査で見つけた重要な情報1}}
- {{調査で見つけた重要な情報2}}
- {{調査で見つけた重要な情報3}}

## 重視するポイント

- {{ユーザーが重視したいポイント1}}
- {{ユーザーが重視したいポイント2}}

## 作成日
{{YYYY-MM-DD}}
```

### ステップ6: ファイルの保存

作成したテーマファイルを以下のパスに保存してください。

**保存先**: `/Volumes/WDBLACK_2TB/Git/sns-content-automation/custom_themes/current_theme.md`

**ファイル名**: `current_theme.md`（常にこの名前で上書き保存）

**バックアップ**: 既存の`current_theme.md`がある場合は、以下の名前でバックアップを作成してください。
- バックアップ名: `YYYYMMDD_HHMMSS_theme_backup.md`

### ステップ7: 完了報告

ファイルを保存したら、以下を報告してください。

```
✅ テーマファイルの作成が完了しました！

【保存先】
/Volumes/WDBLACK_2TB/Git/sns-content-automation/custom_themes/current_theme.md

【メインテーマ】
{{メインテーマ}}

【記事数】
{{作成予定の記事数}}本

【次のステップ】
記事を生成するには、以下のコマンドを実行してください：
「custom-theme-articleスキルを使って、記事を生成してください」
```

## 補足: 対話のトーン

- フレンドリーで親しみやすいトーンで対話してください
- ユーザーの意図を正確に理解するため、必要に応じて質問を追加してください
- 提案は具体的で、実践的な内容にしてください
