# メディアファイル管理ディレクトリ

このディレクトリでは、記事に使用するすべてのメディアファイルを管理します。

## 📁 ディレクトリ構造

- `infographics/`: インフォグラフィック(記事のメインビジュアル)
- `images/`: 画面キャプチャやその他の画像素材
- `designs/`: CANVAで作成したトップ画像やデザインファイル

## 📝 ファイル命名規則

### インフォグラフィック
```
infographics/infographic-XXX.png
```
XXXは記事ID

例: `infographics/infographic-001.png`

### 画像素材
```
images/image-XXX-01.png
images/image-XXX-02.png
```
XXXは記事ID、01/02は連番

例: `images/image-001-01.png`

### デザインファイル
```
designs/top-image-XXX.png
designs/canva-XXX.png
```

例: `designs/top-image-001.png`

## 🎨 デザインガイドライン

### インフォグラフィック
- **推奨サイズ**: 1200x630px (SNS最適サイズ)
- **ファイル形式**: PNG (透過背景の場合) または JPG
- **カラーパレット**: [統一する色を定義]
- **フォント**: [統一するフォントを定義]
- **キャラクター**: [統一するキャラクターを定義]

### トップ画像(Note用)
- **推奨サイズ**: 1280x670px (Note推奨サイズ)
- **ファイル形式**: PNG または JPG
- **ブランディング**: ロゴやキャッチフレーズを含める

## 📊 ファイル管理のベストプラクティス

1. **命名規則の厳守**: 記事IDを必ず含める
2. **適切な圧縮**: ファイルサイズを最適化(100-500KB目安)
3. **バックアップ**: 重要なデザインファイルはCANVAでも保存
4. **整理整頓**: 使用しなくなったファイルは定期的に削除

## 🔗 関連ファイル

各メディアファイルは、対応する記事ファイル(`articles/`)で参照されます。
